## Prerequisites

Before you begin, make sure you have the following installed:

| Tool | Minimum version | Check |
|---|---|---|
| Python | 3.10+ | `python --version` |
| Node.js | 18+ | `node --version` |
| Git | Any recent version | `git --version` |

## Set up the backend

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Rafael-Lee1/healthia-diagnostico-inteligente.git
    cd healthia-diagnostico-inteligente
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    <CodeGroup>

    ```bash macOS / Linux
    cd HEALTHIA
    python -m venv .venv
    source .venv/bin/activate
    ```

    ```powershell Windows
    cd HEALTHIA
    python -m venv .venv
    .venv\Scripts\Activate.ps1
    ```

    </CodeGroup>

    <Tip>Your shell prompt will change to show `(.venv)` when the environment is active.</Tip>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This installs FastAPI, Uvicorn, pandas, NumPy, scikit-learn, and Joblib.
  </Step>

  <Step title="Start the API server">
    ```bash
    uvicorn main:app --reload
    ```

    The API will be available at `http://127.0.0.1:8000`. The `--reload` flag restarts the server automatically when you change source files.
  </Step>
</Steps>

### Verify the backend

Open `http://127.0.0.1:8000` in your browser. You should see:

```json
{
  "message": "Welcome to HealthIA API",
  "mode": "educational"
}
```

The interactive Swagger UI is at `http://127.0.0.1:8000/docs`.

## Set up the frontend

Open a new terminal window. Leave the backend server running.

<Steps>
  <Step title="Navigate to the frontend directory">
    ```bash
    cd frontend
    ```

    Run this from the repository root (the `healthia-diagnostico-inteligente` directory).
  </Step>

  <Step title="Configure the environment">
    Copy the example environment file and confirm the API URL:

    ```bash
    cp .env.example .env
    ```

    The default `.env` contains:

    ```bash .env
    VITE_API_BASE_URL=http://127.0.0.1:8000
    ```

    <Note>
      Change `VITE_API_BASE_URL` if your backend is running on a different host or port. Vite only exposes variables prefixed with `VITE_` to the browser bundle.
    </Note>
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    Vite will print a local URL — typically `http://localhost:5173`.
  </Step>
</Steps>

### Verify the frontend

Open `http://localhost:5173` in your browser. The HealthIA interface should load with the symptom input form.

## Test the API with curl

Send a POST request to the `/predict/` endpoint with a symptom description:

```bash
curl -X POST http://127.0.0.1:8000/predict/ \
  -H "Content-Type: application/json" \
  -d '{
    "sintomas": "febre alta, dor de cabeça intensa, rigidez no pescoço",
    "top_n": 5,
    "patient_context": null
  }'
```

A successful response looks like this:

```json
{
  "input_text": "febre alta, dor de cabeça intensa, rigidez no pescoço",
  "normalized_text": "febre alta dor cabeca intensa rigidez pescoco",
  "top_predictions": [
    {
      "disease": "Meningite",
      "confidence": 0.84,
      "severity": "high",
      "emergency": true
    }
  ],
  "warning": null,
  "disclaimer": "Este resultado é apenas educacional e não substitui avaliação médica profissional.",
  "patient_context": null,
  "top_diagnosis_explanation": "Febre alta combinada com dor de cabeça intensa e rigidez nucal são sintomas clássicos de meningite."
}
```

<Tip>
  You can also include patient context to improve prediction relevance. Pass an object for `patient_context` with fields like `age`, `sex`, `comorbidities`, and `medications`. See the [API reference](/api/predict-post) for the full schema.
</Tip>
